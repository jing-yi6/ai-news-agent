# 🤖 AI News Agent

每日 AI 资讯摘要生成器，自动从 X (Twitter) 获取 AI 相关推文，智能过滤分类，生成精美的 Markdown 日报。

## ✨ 功能特点

- **🐦 多源获取**: 支持获取关注列表或指定用户的推文
- **🎯 智能过滤**: 基于关键词的 AI 内容识别，自动排除无关推文
- **📂 自动分类**: 使用 LLM 将推文智能分类为模型发布、研究进展、产品更新、教程资源等
- **🤖 LLM 摘要**: 支持 OpenAI、兼容格式 API、Anthropic Claude 生成智能摘要
- **📝 Markdown 输出**: 生成美观的日报，包含目录、互动数据、原文链接
- **🔧 插件架构**: 支持自定义数据源和 LLM Provider，易于扩展
- **🚀 无需 API Key**: 使用 twscrape 抓取，无需申请 X API

## 🔑 配置说明

### X (Twitter) 账号配置（必需）

由于使用 twscrape 抓取推文，需要提供 X 账号信息：

#### 方式一：使用 Cookies（推荐）

从浏览器开发者工具复制 cookies：

```bash
X_COOKIES=auth_token=xxx;ct0=yyy
```

获取方法：
1. 登录 X (Twitter) 网页版
2. 打开浏览器开发者工具（F12）
3. 切换到 Application/Storage 标签
4. 找到 Cookies，复制 `auth_token` 和 `ct0` 的值

#### 方式二：使用用户名密码

```bash
X_USERNAME=your_twitter_username
X_PASSWORD=your_twitter_password
X_EMAIL=your_email@example.com
X_EMAIL_PASSWORD=your_email_password
```

### LLM 配置（可选，用于智能分类）

#### 方式一：统一配置格式（推荐）

```bash
LLM_PROVIDER=openai          # openai 或 anthropic
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-3.5-turbo
```

#### 方式二：标准 OpenAI（旧格式，仍支持）

```bash
OPENAI_API_KEY=sk-xxx
OPENAI_BASE_URL=https://api.openai.com/v1  # 可选
OPENAI_MODEL=gpt-3.5-turbo
```

#### 方式三：Anthropic Claude（旧格式，仍支持）

```bash
ANTHROPIC_API_KEY=sk-ant-xxx
ANTHROPIC_MODEL=claude-3-haiku-20240307
```

### 配置示例

**使用 Azure OpenAI：**
```bash
LLM_PROVIDER=openai
LLM_API_KEY=your-azure-key
LLM_BASE_URL=https://your-resource.openai.azure.com/openai/deployments/your-deployment

```

**使用本地模型（Ollama）：**
```bash

LLM_API_KEY=ollama

```

**使用 OpenRouter：**
```bash

LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=anthropic/claude-3-haiku
```

## 🚀 使用方法

### 获取指定用户的推文

```bash
python run.py --users OpenAI DeepMind AnthropicAI
```

### 获取关注列表的推文

```bash
# 设置环境变量 X_USER_ID，或使用 --user-id 参数
python run.py --user-id <your_user_id>
```

### 使用 LLM 智能分类

```bash
python run.py --users OpenAI --use-llm
```

### 使用模拟数据测试

```bash
python run.py --mock
```

### 指定数据源

```bash
# 使用模拟数据源
python run.py --datasource mock --users OpenAI

# 使用 X 数据源（默认）
python run.py --datasource x --users OpenAI
```

### 指定日期（用于测试）

```bash
python run.py --mock --date 2024-01-15
```

### 完整命令行参数

```bash
python run.py --help
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--users` | 指定用户名列表（不含 @） | - |
| `--user-id` | 指定用户 ID（用于获取关注列表） | - |
| `--mock` | 使用模拟数据测试 | False |
| `--datasource` | 数据源类型 (x, mock) | 从环境变量读取 |
| `--date` | 指定日期 (YYYY-MM-DD) | 当天 |
| `--output` | 输出目录 | `output` |
| `--max-following` | 最多获取关注用户数 | 30 |
| `--tweets-per-user` | 每个用户获取推文数 | 30 |
| `--min-engagement` | 最小互动数过滤 | 0 |
| `--use-llm` | 使用 LLM 智能分类 | False |

## 🏗️ 插件架构

### 添加新的 LLM Provider

继承 `BaseLLMProvider` 并实现 `chat_complete()` 方法：

```python
from providers.base import BaseLLMProvider, Message, LLMResponse

class MyProvider(BaseLLMProvider):
    def is_available(self) -> bool:
        try:
            import my_llm_sdk
            return True
        except ImportError:
            return False

    def _init_client(self):
        import my_llm_sdk
        self._client = my_llm_sdk.Client(self.api_key)

    def chat_complete(self, messages: list[Message], **kwargs) -> LLMResponse:
        # 实现调用逻辑
        response = self._client.chat(...)
        return LLMResponse(
            content=response.text,
            usage={"prompt_tokens": ..., "completion_tokens": ...},
            model=self.model
        )
```

然后在 `providers/__init__.py` 注册：

```python
PROVIDER_REGISTRY = {
    "openai": OpenAIProvider,
    "anthropic": AnthropicProvider,
    "myprovider": MyProvider,  # 添加新的 Provider
}
```

## 🎯 分类规则

推文会被自动分类到以下类别：

| 类别 | 说明 |
|------|------|
| **模型发布** | 新模型发布、模型版本更新、新功能上线 |
| **研究进展** | 论文发表、技术突破、benchmark 结果、学术研究 |
| **产品更新** | 产品功能更新、UI改进、新特性上线、Beta版本 |
| **教程资源** | 教程、指南、学习资源、文档、代码示例 |
| **行业动态** | 融资、投资、收购、合作、招聘、公司动态 |
| **其他资讯** | 不属于以上类别的AI相关内容 |

**分类方式**：
- 默认使用关键词匹配进行快速分类
- 启用 `--use-llm` 后使用 LLM 进行智能分类

## ⚠️ 注意事项

1. **账号安全**：使用 Cookies 方式更安全，不会暴露密码
2. **用户 ID**：获取关注列表需要目标用户的数字 ID，非用户名
3. **LLM 可选**：不配置 LLM 也能运行，会使用关键词分类
4. **速率限制**：twscrape 会自动处理速率限制，但频繁抓取可能导致账号暂时受限
5. **插件向后兼容**：新架构保持与旧代码的兼容性，可以逐步迁移

## 🔧 故障排除

### 问题：未找到命令 `conda`

**解决**：安装 [Miniconda](https://docs.conda.io/en/latest/miniconda.html) 或 [Anaconda](https://www.anaconda.com/download)

### 问题：提示缺少模块

**解决**：确保已激活 conda 环境
```bash
conda activate ai-news-agent
```

### 问题：未设置 X 账号信息

**解决**：复制 `.env.example` 为 `.env`，填入你的 X 账号 Cookies 或用户名密码
```bash
cp .env.example .env
# 编辑 .env 文件
```

### 问题：获取关注列表失败

**解决**：确保 `X_USER_ID` 是数字 ID（如 `1234567890`），非用户名

### 问题：LLM 分类不可用

**解决**：确保配置了 LLM API Key 并使用 `--use-llm` 参数：
```bash
python run.py --users OpenAI --use-llm
```

### 问题：ImportError: cannot import name 'xxx'

**解决**：确保在项目根目录运行，且 Python 版本 >= 3.9
```bash
cd ai-news-agent
python run.py --mock
```

### 问题：twscrape 登录失败

**解决**：
1. 检查账号信息是否正确
2. 尝试使用 Cookies 方式登录
3. 如果是新账号，可能需要在网页版完成验证
4. 检查账号是否被临时限制
