# 🤖 AI News Agent

每日 AI 资讯摘要生成器，自动从 X (Twitter) 获取 AI 相关推文，智能过滤分类，生成精美的 Markdown 日报。

## ✨ 功能特点

- **🐦 多源获取**: 支持获取关注列表或指定用户的推文
- **🎯 智能过滤**: 基于关键词的 AI 内容识别，自动排除无关推文
- **📂 自动分类**: 将推文分类为模型发布、研究进展、产品更新、教程资源等
- **🤖 LLM 摘要**: 支持 OpenAI、兼容格式 API、Anthropic Claude 生成智能摘要
- **📝 Markdown 输出**: 生成美观的日报，包含目录、互动数据、原文链接
- **⚡ 速率限制处理**: 自动处理 X API 速率限制
- **🔧 灵活配置**: 支持多种 LLM 后端（OpenAI、Azure、本地模型等）

## 📦 安装

### 方式一：使用 Conda（推荐）

#### 1. 克隆仓库

```bash
git clone https://github.com/jing-yi6/ai-news-agent.git
cd ai-news-agent
```

#### 2. 创建 Conda 环境

```bash
# 从 environment.yml 创建环境
conda env create -f environment.yml

# 激活环境
conda activate ai-news-agent
```

#### 3. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 文件，填入你的 API 密钥
```

#### 4. 验证安装

```bash
python run.py --mock
```

## 🔑 配置说明

### 必需配置

| 环境变量 | 说明 | 获取方式 |
|----------|------|----------|
| `X_BEARER_TOKEN` | X API Bearer Token | [X Developer Portal](https://developer.twitter.com/en/portal/dashboard) |

### 可选配置

#### 方式一：OpenAI 兼容 API（推荐用于第三方服务）

```bash
OPENAI_COMPATIBLE_API_KEY=your_key
OPENAI_COMPATIBLE_BASE_URL=https://api.example.com/v1
OPENAI_COMPATIBLE_MODEL=gpt-3.5-turbo
```

#### 方式二：标准 OpenAI

```bash
OPENAI_API_KEY=sk-xxx
OPENAI_BASE_URL=https://api.openai.com/v1  # 可选，用于代理
OPENAI_MODEL=gpt-3.5-turbo  # 可选
```

#### 方式三：Anthropic Claude

```bash
ANTHROPIC_API_KEY=sk-ant-xxx
```

### 配置示例

**使用 Azure OpenAI：**
```bash
OPENAI_COMPATIBLE_API_KEY=your-azure-key
OPENAI_COMPATIBLE_BASE_URL=https://your-resource.openai.azure.com/openai/deployments/your-deployment
OPENAI_COMPATIBLE_MODEL=gpt-35-turbo
```

**使用本地模型（Ollama）：**
```bash
OPENAI_COMPATIBLE_API_KEY=ollama
OPENAI_COMPATIBLE_BASE_URL=http://localhost:11434/v1
OPENAI_COMPATIBLE_MODEL=llama2
```

**使用 OpenRouter：**
```bash
OPENAI_COMPATIBLE_API_KEY=sk-or-xxx
OPENAI_COMPATIBLE_BASE_URL=https://openrouter.ai/api/v1
OPENAI_COMPATIBLE_MODEL=anthropic/claude-3-haiku
```

## 🚀 使用方法

### 获取指定用户的推文

```bash
python run.py --users OpenAI DeepMind AnthropicAI
```

### 获取关注列表的推文

```bash
# 设置环境变量 X_USER_ID，或使用 --user-id 参数
python run.py --user-id <your_user_id>
```

### 使用 LLM 生成摘要

```bash
python run.py --users OpenAI --use-llm
```

### 使用模拟数据测试

```bash
python run.py --mock
```

### 指定日期（用于测试）

```bash
python run.py --mock --date 2024-01-15
```

### 完整命令行参数

```bash
python run.py --help
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--users` | 指定用户名列表（不含 @） | - |
| `--user-id` | 指定用户 ID（用于获取关注列表） | - |
| `--mock` | 使用模拟数据测试 | False |
| `--date` | 指定日期 (YYYY-MM-DD) | 当天 |
| `--output` | 输出目录 | `output` |
| `--max-following` | 最多获取关注用户数 | 30 |
| `--tweets-per-user` | 每个用户获取推文数 | 5 |
| `--min-engagement` | 最小互动数过滤 | 0 |
| `--use-llm` | 使用 LLM 生成摘要 | False |

## 📁 项目结构

```
ai-news-agent/
├── run.py              # 主入口程序
├── environment.yml     # Conda 环境配置
├── requirements.txt    # pip 依赖
├── .env.example        # 环境变量模板
├── README.md           # 本文档
├── output/             # 输出目录
│   ├── 2024-01-15.md
│   └── 2024-01-15_quick_view.md
└── src/                # 源代码模块
    ├── x_client.py     # X API v2 客户端
    ├── fetcher.py      # 推文获取逻辑
    ├── filter.py       # 内容过滤与分类
    ├── summarizer.py   # 摘要生成（支持多种 LLM）
    └── formatter.py    # Markdown 格式化输出
```

### 模块说明

| 模块 | 功能 |
|------|------|
| `x_client.py` | X API v2 客户端，处理认证和速率限制 |
| `fetcher.py` | 获取推文，支持关注列表和指定用户 |
| `filter.py` | 关键词过滤、互动数筛选、自动分类 |
| `summarizer.py` | LLM 摘要生成，支持 OpenAI/兼容 API/Anthropic |
| `formatter.py` | Markdown 格式化，生成美观的日报 |

## 🎯 分类规则

推文会被自动分类到以下类别：

| 类别 | 触发关键词 |
|------|------------|
| **模型发布** | announcing, introducing, release, launch, new model |
| **研究进展** | paper, research, arxiv, study, benchmark, sota |
| **产品更新** | update, feature, new in, now available, beta |
| **教程资源** | tutorial, guide, how to, learn, course, documentation |
| **行业动态** | funding, investment, acquisition, partnership, hiring |
| **其他资讯** | 默认分类 |

## ⚠️ 注意事项

1. **X API 限制**：免费层有速率限制，如需大量获取请考虑升级
2. **用户 ID**：获取关注列表需要目标用户的数字 ID，非用户名
3. **LLM 可选**：不配置 LLM 也能运行，会使用简单的本地摘要

## 🔧 故障排除

### 问题：未找到命令 `conda`

**解决**：安装 [Miniconda](https://docs.conda.io/en/latest/miniconda.html) 或 [Anaconda](https://www.anaconda.com/download)

### 问题：提示缺少模块

**解决**：确保已激活 conda 环境
```bash
conda activate ai-news-agent
```

### 问题：未设置 X_BEARER_TOKEN

**解决**：复制 `.env.example` 为 `.env`，填入你的 X API Bearer Token

### 问题：获取关注列表失败

**解决**：确保 `X_USER_ID` 是数字 ID（如 `1234567890`），非用户名

### 问题：LLM 摘要不可用

**解决**：确保在 `environment.yml` 中配置了 LLM 依赖，然后重新创建环境：
```bash
conda env update -f environment.yml
conda activate ai-news-agent
```

或手动安装：
```bash
pip install openai anthropic
```
